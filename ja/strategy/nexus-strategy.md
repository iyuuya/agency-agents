## 1. 戦略的基盤

### 1.1 NEXUSが解決する問題

個々のエージェントは強力です。しかし調整なしでは以下の問題が生じます:
- 相反するアーキテクチャ上の決定
- 部門間での重複した作業
- 引き継ぎ境界での品質ギャップ
- 共有コンテキストや組織記憶の欠如

**NEXUSはこれらの失敗モードを排除します**。以下を定義することで:
- **誰が**各フェーズで起動するか
- **何を**生産し、誰のために行うか
- **いつ**引き継ぎ、誰に渡すか
- **どのように**品質が進捗前に検証されるか
- **なぜ**各エージェントがパイプラインに存在するか（無駄なエージェントは不要）

### 1.2 コア原則

| 原則 | 説明 |
|-----------|-------------|
| **パイプラインの整合性** | 品質ゲートを通過しなければフェーズは進まない |
| **コンテキストの継続性** | すべての引き継ぎは完全なコンテキストを持つ — エージェントはコールドスタートしない |
| **並行実行** | 独立したワークストリームをタイムライン短縮のために並行して実行する |
| **主張よりも証拠** | すべての品質評価には証明が必要であり、主張では不十分 |
| **失敗は早く、修正は早く** | タスクあたり最大3回の再試行の後にエスカレーション |
| **信頼できる単一の情報源** | 1つの標準仕様、1つのタスクリスト、1つのアーキテクチャドキュメント |

### 1.3 部門別エージェントロスター

| 部門 | エージェント | NEXUSにおける主な役割 |
|----------|--------|---|
| **エンジニアリング** | Frontend Developer, Backend Architect, Mobile App Builder, AI Engineer, DevOps Automator, Rapid Prototyper, Senior Developer | すべての技術システムの構築、デプロイ、維持 |
| **デザイン** | UI Designer, UX Researcher, UX Architect, Brand Guardian, Visual Storyteller, Whimsy Injector, Image Prompt Engineer | ビジュアルアイデンティティ、ユーザー体験、ブランドの一貫性を定義 |
| **マーケティング** | Growth Hacker, Content Creator, Twitter Engager, TikTok Strategist, Instagram Curator, Reddit Community Builder, App Store Optimizer, Social Media Strategist | 獲得、エンゲージメント、市場プレゼンスを推進 |
| **プロダクト** | Sprint Prioritizer, Trend Researcher, Feedback Synthesizer | 何を、いつ、なぜ構築するかを定義 |
| **プロジェクト管理** | Studio Producer, Project Shepherd, Studio Operations, Experiment Tracker, Senior Project Manager | タイムライン、リソース、部門横断の調整をオーケストレーション |

```

└── Sprint Prioritizer → RICE-scored backlog with sprint assignments
    ├── Input: Task List + Architecture Spec + Budget Framework
    ├── Output: Prioritized sprint plan with dependency map
    └── Validation: Studio Producer confirms strategic alignment
```

## 付録C: NEXUSグロッサリー

| 用語 | 定義 |
|------|-----------|
| **NEXUS** | Network of EXperts, Unified in Strategy（専門家のネットワーク、戦略における統一） |
| **品質ゲート** | 証拠ベースの承認が必要なフェーズ間の必須チェックポイント |
| **Dev↔QAループ** | 各タスクが進む前にQAをパスする必要がある継続的な開発テストサイクル |
| **引き継ぎ** | エージェント間での作業とコンテキストの構造化された転送 |
| **ゲートキーパー** | フェーズの進捗を承認または拒否する権限を持つエージェント |
| **エスカレーション** | 再試行が尽きた後のブロックされたタスクを上位権限にルーティングすること |
| **NEXUS-Full** | すべてのエージェントによる完全なパイプライン起動 |
| **NEXUS-Sprint** | 機能/MVP作業のための15〜25エージェントによる集中パイプライン |
| **NEXUS-Micro** | 特定のタスクのための5〜10エージェントの的を絞った起動 |
| **パイプラインの整合性** | 品質ゲートを通過しなければフェーズは進まないという原則 |
| **コンテキストの継続性** | すべての引き継ぎが完全なコンテキストを持つという原則 |
| **主張よりも証拠** | 品質評価には証明が必要であり、主張では不十分という原則 |
